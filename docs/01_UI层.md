## 二、硬性约束

### 2.1 禁止依赖

```
❌ ui/design_system/kit_* → provider / get_it（任何 import）
❌ ui/design_system/kit_* → domain/usecases（直接调用）
❌ ui/ → infrastructure/（任何方向）
❌ ui/ → shared/constants/infra_constants
❌ 任何组件 → domain/usecases（直接调用）
```

### 2.2 允许依赖

```
✅ ui/ → application/（通过 Provider/Consumer 读取状态）
✅ ui/ → shared/（工具类）
✅ ui/ → shared/constants/ui_tokens、design_system/constants
```

### 2.3 ThemeExtension 唯一性

**ThemeExtension 是 UI 层唯一的跨层渲染 DI 机制。**  
组件通过 `Theme.of(context).extension<AppTheme>()` 获取主题数据，**禁止**直接 import ThemeCatalog 或 GameProvider 做主题计算。

### 2.4 主题消费规范

- RootPage 仅保留**一处** `ValueListenableBuilder<AppTheme>`
- 背景层、Header、胶囊层均从同一 `AppTheme` 实例读取
- 主题计算由 `RootThemeCoordinator` / `RootThemeScope` 单点输出

## 三、strings 拆分规范

| 类型 | 存放位置 | 引用方 |
|---|---|---|
| UI 展示文本（按钮、提示） | `ui/design_system/constants/strings.dart` | ui 层 |
| 业务/日志语义文本 | `shared/constants/domain_constants.dart` | application、domain |

**禁止** application 层 import `ui/design_system/constants/`。

## 四、扩展规范

### 4.1 新增主题皮肤

1. 在 `theme/domain_theme/` 或 `theme/special_theme/` 新增 `@GameTheme()` 类
2. 运行 `tool/theme_registry_gen.dart` 生成 `theme_catalog.g.dart`
3. 不修改 `theme_catalog.dart` 手写区

### 4.2 新增页面

1. 在 `pages/` 下新建目录，如 `pages/new_feature/`
2. 页面通过 `context.read<XxxController>()` 或 `Consumer<XxxController>` 获取状态
3. 不得在页面内 `getIt<>()`
