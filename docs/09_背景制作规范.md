# 背景制作规范

> 本规范用于**新背景的制作与既有背景的修改**。所有主题的 `buildBackground` 实现应遵守本文档；其中**手机与平板分离、两档切档**为必守规则，供后续制作新背景时统一执行。

---

## 一、核心原则

**"层级分离、锚点固定、手机平板两档"**

1. **层级分离**：背景拆为独立的功能层（L0–L4），每层有唯一的定位策略与职责，层序固定。
2. **锚点固定**：每层元素锚定在视口的几何特征点（中心、四角、边缘）上，**禁止**使用 `screenWidth * ratio` 做连续缩放。
3. **手机与平板两档**：通过宽度断点区分**手机（compact）**与**平板（expanded）**，每层尺寸与布局按两档分别给出固定值，不在两档之间做连续缩放。
4. **溢出**：需要超出视口的元素直接溢出，不需做裁切或精确计算溢出量。

---

## 二、手机与平板分离

### 2.1 为何分离

- 官网（如 CiRCLE、DX、Verse）均采用 **SP（手机） / PC（平板）** 两套布局，对应不同视口与视觉权重。
- 背景为纯装饰，中间态（如 400–600dp）人眼差异小，**仅做两档**即可：手机一档、平板一档。

### 2.2 断点唯一定义

| 档位 | 宽度范围 | 典型设备 | 语义 |
|------|----------|----------|------|
| **compact（手机）** | `width < 600` dp | 手机竖屏 | 单栏、浮层导航、背景按手机参数渲染 |
| **expanded（平板）** | `width ≥ 600` dp | 平板、桌面、手机横屏 | 双栏/侧边栏、背景按平板参数渲染 |

**判定方式**：在 `buildBackground` 内部、方法顶部统一判定，不 elsewhere 硬编码等价断点。

```dart
final w = MediaQuery.sizeOf(context).width;
final h = MediaQuery.sizeOf(context).height;
final bool isCompact = w < 600;
```

背景层位于 `PageShell` 内、`ResponsiveLayoutScope` 之上，属于 `ui/` 层闭环，不依赖 `application/`，不违反架构红线。

### 2.3 两档的体现

- **同一套 `buildBackground`**：入口只有一个，内部用 `isCompact` 分支决定各层尺寸与布局。
- **每层两套参数**：L2/L3/L4 的宽高、位置等，均以 `isCompact ? compact值 : expanded值` 形式给出，**禁止**在分支外使用 `w * ratio` 做连续缩放。
- **L0/L1**：可为全尺寸通用（如 L0 纯色、L1 以 `max(w,h)` 为边长），不强制分档；若需分档亦可。

## 三、层级模型

每个主题的 `buildBackground` 返回 `Stack(fit: StackFit.expand)`，自底向上**最多**包含以下 5 层；不是每层都必须存在，但**层序固定**：

| 层序 | 名称 | 职责 | 定位策略 | 是否必须两档 |
|------|------|------|----------|--------------|
| L0 | 底色层 | 纯色或渐变填充 | `Positioned.fill` | 可不分档 |
| L1 | 纹理层 | 可平铺或旋转的满屏花纹 | 视口中心锚定，正方形 | 可不分档（边长取 max(w,h)） |
| L2 | 焦点层 | 主视觉焦点（圆环、地球等） | 视口中心或边缘锚定 | **必须两档** |
| L3 | 边角层 | 四角或边缘的静态装饰 | 四角锚定 | **必须两档** |
| L4 | 氛围层 | 漂浮小装饰（星星、云朵等） | 视口百分比定位 | **必须两档** |

## 四、各层制作规范

### 4.1 L0 底色层

**作用**：填满视口的纯色或渐变，作为最底层。

**要求**：
- 无固定像素值，天然适配所有尺寸；可不分档。
- 渐变/色值为主题品牌固定色，不受 `dark/light` token 影响。

**示例**：
```dart
Positioned.fill(
  child: Container(
    decoration: BoxDecoration(
      gradient: RadialGradient(...), // 或 color: Color(0xFF...)
    ),
  ),
)
```

---

### 4.2 L1 纹理层

**作用**：覆盖在底色上的花纹/纹理，可低速旋转或平铺。

**要求**：
- 以视口中心为锚点的正方形，边长取 `max(w, h)`，保证横竖屏无留白。
- 图片用 `BoxFit.contain` 或 `ImageRepeat.repeat`；旋转建议 ≥ 120 秒/圈。
- 可不分档（尺寸已由长边决定）。

**示例**：
```dart
Positioned.fill(
  child: Center(
    child: SizedBox(
      width: max(w, h),
      height: max(w, h),
      child: Transform.rotate(
        angle: animation,
        child: Image.asset(..., fit: BoxFit.contain),
      ),
    ),
  ),
)
```

---

### 4.3 L2 焦点层

**作用**：主视觉焦点（圆环、地球、大型图案等）。

**要求**：
- **必须两档**：`width/height` 使用 `isCompact ? compactSize : expandedSize`，禁止 `screenWidth * ratio` 或裸 `Transform.scale` 充当缩放。
- 锚点二选一：中心型（`Positioned.fill` + `Center`）或边缘型（如 `Positioned(bottom: offset)` + 水平居中）。
- 允许直接溢出视口；动效参数（速度、方向）可两档一致。

**示例**：
```dart
final double size = isCompact ? 200.0 : 420.0;
Positioned.fill(
  child: Center(
    child: SizedBox(
      width: size,
      height: size,
      child: Image.asset(..., fit: BoxFit.contain),
    ),
  ),
)
```

---

### 4.4 L3 边角层

**作用**：四角或边缘的静态装饰。

**要求**：
- **必须两档**：宽高用 `isCompact ? compactW : expandedW`；compact 下宽边角可用 `double.infinity` 贴边。
- 不做动效，边角为静态锚定。

**示例**：
```dart
final double width = isCompact ? double.infinity : 845.0;
final double? height = isCompact ? 88.0 : null;
Positioned(top: 0, right: 0,
  child: SizedBox(
    width: width,
    height: height,
    child: Image.asset(..., fit: BoxFit.contain),
  ),
)
```

---

### 4.5 L4 氛围层

**作用**：漂浮小装饰（星星、云朵、3D 物件等）。

**要求**：
- **必须两档**：位置用视口百分比（如 `h * topRatio`），尺寸用 `isCompact ? compactSize : expandedSize`。
- 动效可为旋转、漂移、浮现消隐等；compact 下可减少元素数量（性能）。

**示例**：
```dart
final double size = isCompact ? 80.0 : 160.0;
Positioned(
  top: h * 0.12,
  left: 0, right: 0,
  child: Center(
    child: SizedBox(
      width: size,
      child: Image.asset(..., fit: BoxFit.contain),
    ),
  ),
)
```

## 五、参数表组织规范

在 `buildBackground` **方法顶部**集中声明所有与尺寸/位置相关的两档参数，后续 Widget 树只引用变量，不写裸数字、不用 `screenWidth * ratio`。

```dart
@override
Widget buildBackground(BuildContext context) {
  final w = MediaQuery.sizeOf(context).width;
  final h = MediaQuery.sizeOf(context).height;
  final bool isCompact = w < 600;

  // ── L2 焦点层 ──
  final double focusSize = isCompact ? 200.0 : 420.0;

  // ── L3 边角层 ──
  final double topRightW = isCompact ? w : 845.0;
  final double? topRightH = isCompact ? 88.0 : null;

  // ── L4 氛围层 ──
  final double decorSize = isCompact ? 80.0 : 160.0;

  return Stack(
    fit: StackFit.expand,
    children: [
      // L0 → L1 → L2 → L3 → L4，全部使用上述变量
    ],
  );
}
```

## 六、素材要求

| 要求 | 说明 |
|------|------|
| 最低分辨率 | 至少满足 expanded 档需求，推荐宽边 ≥ 1200px |
| 格式 | 优先 `.webp`（透明）；需极致清晰时可用 `.png` |
| 路径与命名 | 统一放在 `assets/background/{game}/{theme}/`，文件名 `snake_case` |
| 引用方式 | 推荐在 `lib/ui/design_system/constants/assets.dart` 中定义常量（如 `AppAssets.xxx`），主题内引用常量 |
| SP 替换图 | 若 compact 需完全不同构图，可提供单独 SP 图（如 `xxx_sp.webp`） |

## 七、新增主题 Checklist

制作新主题背景时，按以下步骤执行：

| 步骤 | 内容 | 验收标准 |
|------|------|----------|
| 1. 确定层级 | 从 L0–L4 中选定需要的层，画出层级草案 | 草案标明每层内容与是否两档 |
| 2. 准备素材 | 至少提供满足 expanded 档分辨率的素材 | 1200dp 宽度下无可见模糊 |
| 3. 定义参数表 | 为 L2/L3/L4 每个元素填写 compact / expanded 两组值，写在方法顶部 | 无裸数字、无 `w * ratio` |
| 4. 搭建 Widget 树 | 按 L0 → L4 顺序搭建，使用 `isCompact` 与参数表变量 | 层序与规范一致 |
| 5. 验证手机 | 在 &lt; 600dp（如 393dp）下检查 | 与设计稿一致，无异常留白或溢出 |
| 6. 验证平板 | 在 ≥ 600dp（如 1024dp）下检查 | 无模糊、无比例失调 |
| 7. 验证切档 | 在 600dp 附近调整窗口宽度 | 切档无突兀跳变（可接受瞬间重排） |

## 八、与现有架构的关系

| 维度 | 规则 |
|------|------|
| 入口 | `AppTheme.buildBackground(BuildContext context)`，由 `ResolvedThemeBundle` 提供，在 `RootPage` 中传入 `PageShell(backgroundOverride: bundle.buildBackground(context))` |
| 尺寸来源 | `MediaQuery.sizeOf(context)`，不依赖 `ResponsiveLayoutScope` |
| DI 机制 | `ThemeExtension` 为 UI 层唯一跨层渲染 DI，颜色 token 由 `AppTheme` 提供 |
| 禁止事项 | 背景层不引入新 InheritedWidget 或全局状态，不依赖 `application/` 层 |

## 九、当前实现参考

以下为项目中三个主题的**当前实现**，供制作新背景时对照；**新背景应按本规范做手机/平板两档**，以 Verse 为两档参考。

| 主题 | 文件 | 档位 | 说明 |
|------|------|------|------|
| **Verse** | `theme_chu/verse.dart` | 两档（`isCompact = w < 600`） | 符合本规范：参数表在顶部，L0–L4 两档固定像素，可作模板 |
| **Circle** | `theme_mai/circle.dart` | 单档（无 isCompact） | 历史实现，仅手机向单布局；新背景应改为两档 |
| **DX** | `theme_mai/dx.dart` | 单档（比例布局） | 历史实现，使用 `screenWidth * ratio`；新背景应改为两档固定像素 |

**资源**：Circle、Verse 使用 `AppAssets`（`constants/assets.dart`）；DX 使用字符串路径。新主题建议统一使用 `AppAssets`。
