# 业务层规范

> 涉及目录：`lib/domain/`  
> 关联：[00_架构总览](./00_架构总览.md)

---

## 一、当前结构

```
lib/domain/
├── entities/           # MaiMusic、TokenBundle、VpnStatus、VpnStartConfig、SyncResult 等
├── value_objects/      # GameType、TransferMode、DifficultySet
├── repositories/       # AuthRepository、VpnRepository、TransferRepository、MusicLibraryRepository
├── services/           # HtmlRecordParser
├── usecases/
│   ├── transfer/       # verify_tokens、start_import、stop_import、handle_native_log、refresh_lxns_token
│   └── music_data/     # init_music_library、sync_music_library
└── games/
    ├── game_module.dart    # 多游戏抽象
    └── game_registry.dart
```

---

## 二、硬性约束

### 2.1 纯 Dart 禁止

```
❌ domain/ 内任何文件不得 import：
  package:flutter/
  package:drift/
  package:dio/
  dart:ui
  package:flutter/services.dart  (MethodChannel)
```

### 2.2 依赖方向

```
✅ domain/ → shared/
✅ domain/ 声明 repositories 接口，由 infrastructure 实现
❌ domain/ → infrastructure/
❌ domain/ → application/
❌ domain/ → ui/
```

---

## 三、Repository 接口规范

- 接口定义在 `domain/repositories/`
- 实现放在 `infrastructure/storage/repositories_impl/` 或 `infrastructure/` 对应模块
- 接口方法返回 `Result<T, E>` 或 `Stream<T>`，不返回裸 `Map`、裸 `bool?`

---

## 四、UseCase 规范

- 编排业务流程，调用 Repository 接口
- 无状态，可 `@factory` 注册
- 命名：`XxxUsecase`，方法 `execute()` 或语义化方法名

---

## 五、GameModule 扩展规范

### 5.1 新增游戏

1. 实现 `domain/games/game_module.dart` 接口
2. 在 `infrastructure/games/` 下新建模块，如 `chunithm/chunithm_module.dart`
3. 在 `GameRegistry` 中注册新模块
4. 配置类放入 `bootstrap/config/`，由 `AppEnv.getTransferConfig(gameType)` 提供

### 5.2 GameModule 接口

```dart
abstract class GameModule {
  GameType get gameType;
  // 提供该游戏所需的配置、上传路径等
}
```
