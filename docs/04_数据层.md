## 二、数据链路

```
OSS (HTTP GET)
    → OssApiClient
    → OssJsonMapper (List<dynamic> → MaiMusicRow/MaiUtageRow)
    → MaiMusicDao.batchInsert*
    → SQLite (Drift)

SQLite (Drift watchSongs)
    → MaiMusicDao
    → MusicRowMapper (MaiMusicTableData → MaiMusic)
    → MusicLibraryRepositoryImpl
    → Stream<List<MaiMusic>>
    → InitMusicLibraryUsecase
    → MusicLibraryController
    → UI
```

## 三、硬性约束

### 3.1 层级边界

- **row_models**：数据库写入/读取的 DTO，仅限 `infrastructure/storage/sql/`
- **mappers**：Drift 类型 ↔ 领域实体 / row 模型，仅限 `infrastructure/`
- **application 不得直接调用 DAO**：必须经 `MusicLibraryRepository` 等接口

### 3.2 Mapper 位置

- `music_row_mapper`：MaiMusicTableData → MaiMusic，位于 `infrastructure/storage/sql/mappers/`
- `oss_json_mapper`：List<dynamic> → MaiMusicRow/MaiUtageRow，同上

### 3.3 StorageService

- 位于 `infrastructure/storage/secure/`
- 封装 FlutterSecureStorage，提供 `save`、`read`、`delete`
- Key 常量定义在 StorageService 内

## 四、扩展规范

### 4.1 新增表

1. 在 `tables/` 新增 `.dart` 表定义
2. 在 `app_database.dart` 注册表与 DAO
3. 运行 `dart run build_runner build`
4. 在 `row_models/` 新增行模型，`mappers/` 新增 Mapper

### 4.2 新增 Repository 实现

1. 在 `domain/repositories/` 声明接口
2. 在 `infrastructure/storage/repositories_impl/` 实现
3. 在 `bootstrap/di/injection.dart` 注册 `Repository → Impl`
