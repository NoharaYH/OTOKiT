## 一、当前结构

```
lib/infrastructure/network/
├── clients/            # LxnsApiClient、DivingFishApiClient、OssApiClient
├── dto/                # token_response_dto
├── oauth/              # pkce_helper、oauth_callback_server

lib/infrastructure/native/
├── channel/            # vpn_channel_gateway（实现 VpnRepository）
└── models/             # vpn_start_payload
```

## 二、硬性约束

### 2.1 HTTP 客户端

- **禁止** application 直接持有 Dio 或调用 HTTP
- HTTP 调用统一经 `LxnsApiClient`、`DivingFishApiClient`、`OssApiClient` 等
- 配置（baseUrl、clientId 等）通过 `AppEnv` 注入，不硬编码

### 2.2 MethodChannel

- **禁止** application 直接持有 MethodChannel
- VPN 通信经 `VpnChannelGateway` 实现 `VpnRepository`
- application 只依赖 `VpnRepository` 接口，订阅 `statusStream`、`logStream`

### 2.3 OAuth

- PKCE、HttpServer 等放在 `infrastructure/network/oauth/`
- application 经 UseCase 调用，不直接操作 HttpServer

### 2.4 解析器

- 接口定义在 `domain/services/`（如 `HtmlRecordParser`）
- 实现在 `infrastructure/parsers/`（如 `maimai_html_parser_impl.dart`）

## 三、扩展规范

### 3.1 新增 API 平台

1. 在 `infrastructure/network/clients/` 新增 `XxxApiClient`
2. 构造注入 Dio、AppEnv
3. 返回 `Result<T, NetworkException>` 或 DTO，不返回裸 `Map?`

### 3.2 新增 Native Channel

1. 创建 `XxxChannelGateway` 实现对应 Repository 接口
2. 在 `bootstrap/di` 注册 Gateway → Repository
3. application 仅依赖 Repository 接口
