## 二、硬性约束

### 2.1 DI 规则

- **main.dart**：仅调用 `configureDependencies()`，使用 `MultiProvider` 挂载
- **所有 Controller / Provider**：经 GetIt 注册，`ChangeNotifierProvider(create: (_) => getIt<Xxx>())`
- **禁止** application 内 `getIt<>()` 在构造函数内调用（服务定位器反模式）
- **禁止** main 中 `new GameProvider()` 等直接构造（应全部来自 GetIt）

### 2.2 AppEnv 使用

- 抽象定义在 `shared/env/app_env.dart`
- 实现放在 `bootstrap/config/prod_env.dart`
- 仅 `bootstrap`、`infrastructure` 直接依赖 AppEnv；application 不引用 bootstrap/config

### 2.3 注册顺序

- 先注册基础设施（Dio、Storage、AppDatabase、DAO）
- 再注册 Repository 实现 → Repository 接口
- 再注册 UseCase
- 最后注册 Controller

## 三、扩展规范

### 3.1 新增依赖

1. 若使用 `@injectable` / `@lazySingleton`，运行 `dart run build_runner build`
2. 若需手动注册，在 `injection.dart` 的 `configureDependencies` 中追加
3. 确保无重复注册（同类型仅注册一次）
