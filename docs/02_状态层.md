## 一、当前结构

```
lib/application/
├── controllers/       # 概念上：TransferController、MusicLibraryController
├── coordinators/      # RootThemeCoordinator、RootThemeScope
├── bootstrap/         # StartupBootstrap
├── mai/               # mai_music_provider（MusicLibraryController）
├── chu/               # chu_music_provider
├── shared/            # game_provider、navigation_provider、toast_provider
└── transfer/          # transfer_provider（TransferController）、sync_settings_provider
```

## 二、Controller 硬性约束

### 2.1 职责边界

| 属于 Controller | 不属于 Controller |
|---|---|
| 持有 UI State | VPN MethodChannel 初始化 |
| 响应用户操作 | OAuth PKCE、HttpServer |
| 调用 UseCase | URL 拼装、HTML 解析 |
| 订阅 Repository Stream | Token 存取（由 UseCase + Repository） |

### 2.2 行数约束

- `TransferController`：≤ 200 行
- `MusicLibraryController`：≤ 100 行

### 2.3 DI 约束

```
✅ Controller 通过构造函数注入 UseCase、Repository 接口
✅ Controller 通过 GetIt factory 注册
❌ Controller 内部使用 getIt<>()（禁止服务定位器）
```

### 2.4 状态类规范

- 使用 immutable 状态类：`TransferUiState`、`TransferFormState`、`TransferLogState`
- 通过 `copyWith` 更新，`notifyListeners()` 通知 UI

---

## 三、RootThemeCoordinator

- 消费 GameProvider / MaiMusicProvider（或 AppShellController）
- 输出 `ValueNotifier<AppTheme>`
- RootPage 仅保留一处 `ValueListenableBuilder<AppTheme>`

---

## 四、启动页规范

- **StartupBootstrap**：在根入口执行一次 `gp.init()` → `nav.setInitialTag(initialTag)`
- **ScoreSyncPage**：不得调用 `gameProvider.init()` 或 `setInitialTag`，不承担启动页职责
- 启动偏好解析与应用：应用生命周期内只执行一次

---

## 五、扩展规范

### 5.1 新增 Controller

1. 在 `application/controllers/` 或对应模块下新建
2. 使用 `@injectable` 注解，依赖通过构造函数注入
3. 在 `bootstrap/di/injection.dart` 或通过 `@InjectableInit` 自动注册
4. main.dart 中 `ChangeNotifierProvider(create: (_) => getIt<XxxController>())`
