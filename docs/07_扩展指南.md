# OTOKiT 扩展指南

> 本文档描述新增游戏、页面、UseCase 的规范流程，**必须按此执行**。

---

## 一、新增游戏

### 1.1 流程

1. **domain**：在 `domain/value_objects/game_type.dart` 新增枚举值
2. **domain**：在 `domain/repositories/` 中确认是否需要新 Repository 接口
3. **infrastructure**：在 `infrastructure/games/` 下新建模块，实现 `GameModule`
4. **bootstrap**：在 `bootstrap/config/` 新增 `xxx_config.dart`，在 `prod_env.getTransferConfig` 中处理
5. **bootstrap/di**：在 `GameRegistry` 中注册新模块
6. **application**：若有游戏专属 Controller，新建并注册

### 1.2 约束

- 上层（application、ui）不直接 `if (gameType == 1)`，使用 `GameType` 枚举
- 配置统一经 `AppEnv.getTransferConfig(gameType)` 提供

---

## 二、新增页面

### 2.1 流程

1. **ui**：在 `ui/pages/` 下新建目录，如 `new_feature/`
2. **application**：若需新状态，新建 Controller（≤200 行），经 GetIt 注册
3. **main**：在 `MultiProvider` 中挂载新 Controller
4. **navigation**：在 `NavigationProvider` / `PageTag` 中增加路由

### 2.2 约束

- 页面通过 `context.read<XxxController>()` 或 `Consumer` 获取状态
- 禁止在页面内 `getIt<>()` 或 import `get_it`
- 主题消费用 `Theme.of(context).extension<AppTheme>()`

---

## 三、新增 UseCase

### 3.1 流程

1. **domain**：在 `domain/usecases/` 对应子目录新建 `xxx_usecase.dart`
2. **domain**：依赖 Repository 接口，不依赖实现
3. **infrastructure**：确保 Repository 实现已存在
4. **bootstrap/di**：在 `injection.dart` 中 `registerLazySingleton<XxxUsecase>(...)`
5. **application**：Controller 构造函数注入 UseCase

### 3.2 约束

- UseCase 无状态，可 `@factory` 注册
- 返回 `Result<T, E>` 或 `Stream<T>`，不返回裸 `null` / `Map?`

---

## 四、新增主题皮肤

1. 在 `ui/design_system/theme/` 下新建 `@GameTheme()` 类
2. 运行 `dart run build_runner build` 或 `tool/theme_registry_gen.dart`
3. 不修改 `theme_catalog.dart` 手写区

---

## 五、新增 Repository

1. **domain**：在 `domain/repositories/` 声明接口
2. **infrastructure**：在对应 `repositories_impl/` 实现
3. **bootstrap/di**：`getIt.registerLazySingleton<Repo>(() => getIt<RepoImpl>())`
